### 3. Personal Data We Collect

Gumi Save collects only the minimum data necessary to operate the service (data minimization principle).

#### 3.1 Facebook Login Data
When you log in using Facebook, we receive only the following data that you have authorized in your Facebook privacy settings:

| Data | Purpose |
|---|---|
| Profile name (display name) | To identify and display your account |
| Email address | Account creation, login, and service notifications |
| Birthday (date of birth) | Age-appropriate service delivery per Facebook's API |

We do **not** access your Facebook friends list, posts, messages, likes, or any other Facebook data. We do **not** post anything to your Facebook account.

#### 3.2 Household Budget Data
Data you voluntarily enter into the app:

- Income and expense records (amounts, dates, categories)
- Budget envelope allocations and balances
- Savings goals, target amounts, and progress notes
- Transaction descriptions and personal notes

#### 3.3 What We Do NOT Collect

- Precise GPS location
- Device contacts or call logs
- SMS messages
- Bank account numbers, GCash PIN, Maya credentials, or any payment credentials
- Camera access (this app does not include receipt scanning)
- Any data from other apps on your device
- Device identifiers beyond what is required for push notifications

> **Important:** All financial data in Gumi Save is self-reported by you. We do not connect to any bank, remittance service, GCash, Maya, or payment system. No actual money moves through this app.

#### 3.4 Technical Data (Automatically Collected)
- App version number
- Operating system type and version (Android/iOS)
- Crash logs and error reports (anonymized, for bug fixing only)
- Language preference

### 9. Data Security

Our security measures comply with **NPC Circular No. 2023-06** (Minimum Security Requirements):

**Technical Measures:**
- All data in transit is encrypted using TLS 1.2 or higher
- Database storage is encrypted at rest using AES-256
- Authentication tokens are stored securely and rotated regularly
- Strict role-based access controls (RBAC) limit who within KawKaw.dev can access user data

**Organizational Measures:**
- Privacy Impact Assessment (PIA) is conducted before implementing new data processing activities, consistent with NPC guidelines
- Staff handling personal data are trained on data privacy obligations
- Vendor contracts (DPAs) are maintained for all sub-processors

**Incident Management — Personal Data Breach Notification (RA 10173 + NPC Breach Notification Rules):**

In the event of a personal data breach, we will follow this mandatory two-step procedure:

**Step 1 — Within 72 hours of discovery:**
- Notify the NPC via the official **NPC Data Breach Notification Management System (DBNMS)** (submissions outside DBNMS are not valid)
- Notify affected data subjects individually via email
- If the breach involves 100 or more data subjects, or involves sensitive personal information that may harm affected parties, no delay is permitted — notification is immediate

**Step 2 — Within 5 business days:**
- Submit a complete detailed breach report to the NPC via DBNMS containing: nature of the breach, personal data involved, number of affected data subjects, remediation steps taken, and designated representative contact details

**Annual Reporting:** In accordance with NPC requirements, we submit an Annual Security Incident Report (ASIR) via the NPC DBNMS every year, documenting all security incidents including those that did not require immediate breach notification.
